NAME
    ALTER DISTRIBUTION - Modifies distribution metadata.

SYNTAX
    ALTER DISTRIBUTION <distributionID> ATTACH RELATION <relationName>
      DISTRIBUTION KEY <keys> [HASH FUNCTION <hashFunction>]
    ALTER DISTRIBUTION <distributionID> DETACH RELATION <relationName>
    ALTER DISTRIBUTION <distributionID> ALTER RELATION <relationName>
      DISTRIBUTION KEY <keys> [HASH FUNCTION <hashFunction>]
    ALTER DISTRIBUTION <distributionID> ALTER RELATION <relationName>
      SCHEMA <schemaName>
    ALTER DISTRIBUTION <distributionID> ALTER RELATION <relationName>
      RENAME DISTRIBUTION COLUMN <oldColumnName> TO <newColumnName>
    ALTER DISTRIBUTION <distributionID> ADD DEFAULT SHARD <shardID>
    ALTER DISTRIBUTION <distributionID> DROP DEFAULT SHARD

    where keys is a comma-separated list of column names (e.g., "col1, col2")
    where hashFunction is one of: IDENTITY, MURMUR, CITY

DESCRIPTION
    Modifies distribution metadata.

    Covers relation attachments, distribution key metadata, schema metadata,
    and default shard settings. This command operates on SPQR metadata and does
    not execute table DDL on shards.

    ATTACH RELATION attaches one or more relations to the distribution with a
    specified distribution key. For composite sharding keys, specify multiple
    column names separated by commas in the DISTRIBUTION KEY clause.

    DETACH RELATION detaches a relation from the distribution.

    ALTER RELATION DISTRIBUTION KEY replaces the entire distribution key of an
    already attached relation. This overwrites all column names and hash functions
    in one shot — it does not patch individual columns. Use this when you need to
    change the key structure entirely, or to fix expression-based routing entries
    (which RENAME DISTRIBUTION COLUMN does not support). The new key must have the
    same number of entries as the distribution's column types.

    ALTER RELATION SCHEMA changes the schema associated with an attached relation
    in the distribution metadata.

    RENAME DISTRIBUTION COLUMN is a metadata-only fix that renames a column in the
    distribution key of an attached relation. It does not execute ALTER TABLE RENAME
    COLUMN on PostgreSQL — it only updates the column name stored in SPQR metadata.
    The primary use case is fixing a case mismatch between the column name in SPQR
    metadata and the actual PostgreSQL column. For composite keys, call this command
    once per column that needs renaming.

    ADD DEFAULT SHARD sets a default shard for the distribution, routing records that
    do not match any key range. DROP DEFAULT SHARD removes the default shard setting.

WARNINGS
    - ALTER RELATION operations are metadata changes and may affect routing immediately.
    - Use maintenance windows for key/schema metadata changes in write-heavy workloads.

PARAMETERS
    <distributionID> - Identifier of the distribution to modify
    <relationName> - Relation to attach, detach, or alter
    <keys> - Distribution key definition (column list and optional hash functions)
    <hashFunction> - Hash function to apply to the key (IDENTITY, MURMUR, or CITY)
    <schemaName> - Schema metadata value for an attached relation
    <oldColumnName> - Existing distribution key column name in metadata
    <newColumnName> - New distribution key column name in metadata
    <shardID> - Shard identifier to set as default shard

EXAMPLES
    Attach relation with single distribution key:
        ALTER DISTRIBUTION ds1 ATTACH RELATION orders DISTRIBUTION KEY order_id;
    Attach relation with composite key and hash function:
        ALTER DISTRIBUTION ds2 ATTACH RELATION users
            DISTRIBUTION KEY tenant_id, user_id HASH FUNCTION MURMUR;
    Detach relation:
        ALTER DISTRIBUTION ds1 DETACH RELATION orders;
    Replace relation distribution key:
        ALTER DISTRIBUTION ds1 ALTER RELATION orders DISTRIBUTION KEY customer_id;
    Change relation schema:
        ALTER DISTRIBUTION ds1 ALTER RELATION orders SCHEMA my_schema;
    Fix column name casing in metadata (w_id vs W_ID):
        ALTER DISTRIBUTION ds1 ALTER RELATION rc_test RENAME DISTRIBUTION COLUMN W_ID TO w_id;
    Set default shard:
        ALTER DISTRIBUTION ds1 ADD DEFAULT SHARD sh1;
    Remove default shard:
        ALTER DISTRIBUTION ds1 DROP DEFAULT SHARD;

SUBCOMMANDS
    ALTER DISTRIBUTION ATTACH RELATION
    CREATE RELATION
    ALTER DISTRIBUTION DETACH RELATION
    ALTER DISTRIBUTION ALTER RELATION DISTRIBUTION KEY
    ALTER DISTRIBUTION ALTER RELATION SCHEMA
    ALTER DISTRIBUTION RENAME DISTRIBUTION COLUMN
    ALTER DISTRIBUTION ADD DEFAULT SHARD
    ALTER DISTRIBUTION DROP DEFAULT SHARD

TIPS
    - Use CREATE DISTRIBUTION first, then attach relations and create key ranges
    - Verify metadata after changes with SHOW relations and SHOW distributions
    - Prefer replacing full distribution keys when relation uses expression-based routing
    - For composite keys, call RENAME DISTRIBUTION COLUMN once per column that needs renaming
