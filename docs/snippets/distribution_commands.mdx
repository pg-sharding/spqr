### ALTER DISTRIBUTION

Modifies distribution metadata: relation attachments, distribution key metadata,
schema metadata, and default shard settings.

This command operates on SPQR metadata and does not execute table DDL on shards.


<Warning>ALTER RELATION operations are metadata changes and may affect routing immediately.</Warning>

<Warning>Use maintenance windows for key/schema metadata changes in write-heavy workloads.</Warning>


```sql
ALTER DISTRIBUTION <distributionID> ATTACH RELATION <relationName> DISTRIBUTION KEY <keys>
ALTER DISTRIBUTION <distributionID> DETACH RELATION <relationName>
ALTER DISTRIBUTION <distributionID> ALTER RELATION <relationName> DISTRIBUTION KEY <keys>
ALTER DISTRIBUTION <distributionID> ALTER RELATION <relationName> SCHEMA <schemaName>
ALTER DISTRIBUTION <distributionID> ALTER RELATION <relationName>
  RENAME DISTRIBUTION COLUMN <oldColumnName> TO <newColumnName>
ALTER DISTRIBUTION <distributionID> ADD DEFAULT SHARD <shardID>
ALTER DISTRIBUTION <distributionID> DROP DEFAULT SHARD
```

**Examples:**

```sql
-- Attach relation with distribution key
ALTER DISTRIBUTION ds1 ATTACH RELATION orders DISTRIBUTION KEY order_id;
-- Detach relation
ALTER DISTRIBUTION ds1 DETACH RELATION orders;
-- Replace relation distribution key
ALTER DISTRIBUTION ds1 ALTER RELATION orders DISTRIBUTION KEY customer_id;
-- Rename key column metadata
ALTER DISTRIBUTION ds1 ALTER RELATION orders RENAME DISTRIBUTION COLUMN old_id TO new_id;
-- Set default shard
ALTER DISTRIBUTION ds1 ADD DEFAULT SHARD sh1;
```

### CREATE DISTRIBUTION

Creates a sharding distribution with a distribution identifier and key column types.

A distribution defines the sharding key type layout used by attached relations and key ranges.
Optionally, specify a default shard to route records that do not match any key range.


```sql
CREATE DISTRIBUTION <distributionID>
  [COLUMN TYPES <types> | (<types>)]
  [DEFAULT SHARD <shardID>]

where types is a comma-separated list of column types for the distribution key,
and each type is one of: INTEGER, INT, VARCHAR, UUID (VARCHAR/INTEGER may be followed by HASH)
```

**Examples:**

```sql
-- Single integer key distribution
CREATE DISTRIBUTION ds1 COLUMN TYPES integer;
-- Single varchar key distribution with hash
CREATE DISTRIBUTION ds2 COLUMN TYPES varchar hash;
-- Composite key distribution
CREATE DISTRIBUTION ds3 COLUMN TYPES integer, varchar;
-- Parenthesized type syntax with default shard
CREATE DISTRIBUTION ds4 (integer, varchar hash) DEFAULT SHARD sh1;
```

### DROP DISTRIBUTION

Removes a distribution from cluster metadata.

Use CASCADE to remove dependent metadata objects when required.
Use DROP DISTRIBUTION ALL to remove all distributions.


<Warning>Dropping a distribution removes routing metadata and may impact attached relations.</Warning>


```sql
DROP DISTRIBUTION <distributionID> [CASCADE]
DROP DISTRIBUTION ALL [CASCADE]
```

**Examples:**

```sql
-- Drop one distribution
DROP DISTRIBUTION ds1;
-- Drop distribution with dependent metadata
DROP DISTRIBUTION ds1 CASCADE;
-- Drop all distributions with cascade
DROP DISTRIBUTION ALL CASCADE;
```
